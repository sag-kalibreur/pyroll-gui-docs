1. Overview

The Results tab is populated automatically after a successful simulation run from the Pass Design tab. It displays all computed rolling data in structured form: a tabular summary of each roll pass, an engine utilization evaluation, and a comprehensive set of result plots. Data can be exported in multiple formats.

The tab remains empty until at least one sequence has been solved. Results are retained in memory until a new simulation is run or the project is reset. Previously saved projects that include simulation results will restore the results automatically when the project file is loaded.

2. Simulation Configuration

At the top of the Results tab, one or more Simulation Configuration cards are shown — one per sequence. Each card summarises the parameters that governed the simulation.

FieldDescription
Steel GradeMaterial identifier taken from the In Profile tab (e.g. 11SMn30).
Spreading ModelThe spreading law used by PyRolL for width calculation (e.g. Marini Spreading, Wusatowski).
Solve MethodForward Solve, Backward Solve, or Standard Solve — determines the direction of the numerical iteration.
Coulomb Friction CoefficientThe global friction coefficient applied in the simulation.
When multiple sequences are present, each sequence shows its own configuration card side by side. Different sequences can use different spreading models or solve methods within the same project.

3.1 Hot / Cold Switcher

Above the results table, a Hot / Cold toggle button is visible. This switch converts all dimension values (Bar Height, Bar Width, Bar Area, Diameter, Gap) between hot (simulated) and cold (room temperature) dimensions.

The conversion uses the thermal expansion coefficient and the material temperature from the In Profile configuration. Selecting Cold is useful when comparing simulated dimensions to final product drawings which are specified at room temperature.

The Hot/Cold toggle only affects dimensional columns (lengths, widths, areas). Force, torque, temperature, RPM, and ratio columns are not affected.

3.2 Special Case: Finishing Block

When a Finishing Block is part of the rolling schedule, its passes appear in the results table with additional columns and adjusted labels:

  • A Finishing Block header row groups the passes belonging to the block.
  • The Tension After column shows the gear-controlled tension between block passes (as a percentage of flow stress).
  • Bar Width (corrected) and Bar Area (corrected) reflect the geometry after gear-tension correction.
  • Roll Force (corrected), Roll Torque (corrected), and Power (corrected) are calculated with the gear tension taken into account.
  • For block passes the Roll RPM (ideal) and Motor RPM (ideal) represent the tension-free ideal values used for gear ratio setup.

4. Engine Evaluation

Below the Roll Pass Results table, an Engine Evaluation section is shown automatically whenever engines have been defined in the Engine Configuration tab. It compares the motor demands calculated by the simulation against the rated engine capacities.

The table shows the following columns per engine:

ColumnDescription
EngineEngine name as configured in the Engine Configuration tab.
TypMotor type: DC (grey badge) or AC (blue badge).
Assigned StandsThe roll pass labels that are driven by this engine.
Max Engine RPMMaximum mechanical speed at the roll (engine max RPM after gear ratio).
Calculated Engine RPMThe highest RPM required by any assigned stand in the simulation. Highlighted green (OK) or red (OVERLOAD).
Aggregated Required kWSum of power demands from all stands assigned to this engine.
Rated kWRated continuous power of the engine as configured.
Available Power kWDerated power available at the actual operating RPM (AC motor characteristic).
StatusOK if all limits are within bounds, OVERLOAD if any limit is exceeded.
The Engine Evaluation section only appears if engines have been configured. If no engines are defined, this section is hidden. Go to the Engine Configuration tab to set up engines.

4.1 Special Case: Finishing Block

When interstand tensions are present (e.g. from a Finishing Block), the engine evaluation table gains additional columns to reflect the tension-corrected demands:

  • Required kW (corrected) — power demand accounting for the gear-controlled tension between passes.
  • Calculated RPM (corrected) — RPM at the tension-corrected operating point.
  • Available Power kW (corrected) — derated power available at the corrected RPM.
  • A second Status (corrected) column reflects whether the engine is within limits under tension conditions.

This allows direct comparison of the tension-free design point and the actual loaded condition side by side in the same row.

5. Result Plots

All plots are rendered below the Engine Evaluation. They are loaded progressively as the page scrolls to keep the interface responsive for long rolling schedules. Each plot can be exported individually via the Save Results export dialog.

Three plots are only shown when specific pass types are present in the schedule:
Finishing Block Motor Power and Torque Utilization plots
Asymmetric Pass Pillar Elongation & Spread and Sub-Pass Detail plots
Flat Pass Siebel Diagram

5.1 Profile Cross Section Area

Shows the outgoing cross-section area [mm²] for each roll pass as a line/point chart. This is the most direct indicator of the overall reduction along the rolling line — the curve should decrease monotonically from the initial billet area to the final product area.

5.2 Draft & Reduction per Pass

A combined bar/line chart showing two quantities per pass:

  • Draft [mm] (blue bars) — the absolute height reduction (h_in − h_out) per pass.
  • Reduction [%] (red dashed line) — the fractional area reduction as a percentage.

This chart is useful for identifying passes with disproportionately high or low reductions and for verifying that the reduction schedule is well balanced.

5.3 Equivalent Strain

Displays the cumulative equivalent (logarithmic) true strain per pass. The strain accumulates pass by pass and reflects the total deformation history of the workpiece. A high value indicates heavy forming at that pass; a drop may indicate a tension reversal or a finishing block pass with very low reduction.

5.4 Roll Pass Filling Ratios

The filling ratio is the ratio of the actual bar cross-section area to the groove cross-section area: i = A_bar / A_groove.

  • Values close to 1.0 indicate that the bar fills the groove almost completely — good for shape accuracy.
  • Values above 1.0 indicate overfilling, which produces flash and may damage rolls or guides.
  • Values well below 1.0 (e.g. finishing block passes) are intentional and leave space for further forming.

5.5 Temperature Profile

The temperature section contains two sub-plots:

  • Temperature Profile Along Rolling Line — continuous temperature curves (Core, Mean, Surface) plotted against the position [m] along the rolling line. Transport sections are visible as gradual cooling curves; roll passes appear as short drops (surface) and slight rises (core).
  • Temperature Profile per Step — the same three temperatures shown discretely per roll pass and transport step. This format makes it easy to read off exact temperatures at each pass entry/exit.

In a multi-sequence project, the temperature plot spans all sequences in the order they were solved.

5.6 Roll Force

Bar chart showing the total rolling force [kN] for each pass. Higher forces typically occur in the earlier (roughing) passes due to larger cross-sections and higher material flow stress. The chart is useful for sizing roll journals, housings, and screwdown systems.

5.7 Roll Torque

Bar chart showing the torque per roll [kNm] for each pass. Combined with the RPM values from the results table, this allows the required motor power to be cross-checked: P = 2 · π · n/60 · M_roll · (number of rolls).

5.8 Elongation Efficiency & Spread

Combined line chart with two series:

  • Elongation (yellow) — elongation efficiency per pass (actual elongation / ideal elongation). Values below 1 indicate energy lost to lateral spread.
  • Spread (blue dashed) — width increase ratio B_out / B_in. Values above 1.0 indicate that the material is flowing sideways.

Finishing block passes typically show very low elongation efficiency (≈0.3–0.4) and spread close to 1.0, which reflects the constrained deformation mode in small-diameter passing grooves.

5.9 Elongation Ratio & True Strain per Pass

Combined bar/line chart with two series:

  • Elongation Ratio λ (green bars & line) — the length elongation factor per pass: L_out / L_in. A value of 1.25 means the bar is 25 % longer after this pass.
  • True Strain ε (purple dashed line) — the natural logarithm of the elongation ratio: ln(λ). Shown on the right axis.

5.10 In/Out Profile Velocity

Grouped bar chart showing the bar speed [m/s] entering and leaving each roll pass:

  • In Profile (yellow) — entry velocity.
  • Out Profile (blue) — exit velocity.

The velocity increases continuously from stand to stand (volume conservation). This chart is critical for verifying that the RPM schedule is consistent — mismatches lead to interstand tension or compression.

5.11 Motor Power Only: Finishing Block

This plot appears exclusively when at least one Finishing Block is present in the schedule. It shows the demanded motor power [kW] per engine across all finishing block passes, alongside the rated engine power for comparison. The chart makes it easy to identify which engine is closest to its capacity limit.

The chart shows:

  • Required Power bars (yellow) — individual power demand [kW] per finishing block pass.
  • P available (red dashed) — the derated available motor power at the actual engine RPM.
  • P total req. (green dashed) — the total aggregated power demand across all block passes.
  • Engine RPM (orange) — the calculated engine speed annotation.
If no finishing block is configured, this plot is hidden. Configure a finishing block in the Pass Design tab and assign engines in the Engine Configuration tab to activate this plot.

5.12 Torque Utilization Only: Finishing Block

Also only shown when a Finishing Block is present. Displays the torque utilization for each finishing block pass as a percentage of the engine's rated torque capacity. A utilization above 100 % indicates an overload condition for that pass.

The chart shows:

  • Torque Util. / Stand bars (yellow) — torque utilization [%] for each pass in the block.
  • T nominal 100 % (red dashed) — the rated torque limit at 100 %.
  • T total (green dashed) — the overall torque utilization across all block passes combined.
  • Engine RPM (orange) — the calculated engine speed at this operating point.
Both the Motor Power and Torque Utilization plots are rendered directly below the Engine Evaluation section, before the standard Result Plots.

5.13 Pillar Elongation & Spread Distribution Only: Asymmetric Pass

Shown only when at least one Asymmetric Roll Pass is part of the schedule. Plots the local elongation factor λ and spread factor β as a function of the half-width position [mm] for each asymmetric pass. This shows how deformation is distributed across the bar's cross-section — the centre elongates more than the edges.

5.14 Asymmetric Sub-Pass Detail Only: Asymmetric Pass

Also only shown for Asymmetric Roll Passes. Provides a detailed breakdown per pass with:

  • Upper vs. Lower Sub-Pass Comparison — bar chart comparing Roll Force, Roll Torque, Bite Angle, Exit Temperature, Height and Width for the upper and lower groove sub-simulations.
  • Pillar Elongation — Upper vs. Lower Groove (Symmetric Sub-Analysis) — shows the elongation distribution for each symmetric sub-simulation alongside each other, allowing quantification of the asymmetry in deformation.
  • Pillar Elongation tables — numerical tables of pillar elongation values at discrete half-width positions for both the upper and lower groove.

5.15 Siebel Diagram — Constriction / Bulging Only: Flat Pass

Shown only when at least one Flat Pass (Two-Roll Pass with a flat or box groove) is present in the schedule. The Siebel diagram classifies each flat pass as either Bulging or Constriction based on the ratios b₀/h₀ and LD/h₀:

  • b₀/h₀ — ratio of initial bar width to initial bar height.
  • LD/h₀ — ratio of contact arc length to initial bar height.

Each flat pass is plotted as a yellow circle. Points in the Bulging region (upper-left, red background) indicate that the material tends to bulge outward. Points in the Constriction region (lower-right, blue background) indicate lateral constriction. The boundary curve corresponds to the Siebel criterion constant C = 0.355.

Points above the boundary curve are annotated with their distance Δ from the boundary, indicating the magnitude of the tendency to bulge. This is relevant for edge quality and camber prediction.

See the Flat Rolling Edge Deformation – Siebel Criterion document (Help menu) for a detailed mathematical description of the method.

6. Export Options

Click 💾 Save Results to open the export dialog. Enter a custom filename in the input field to the left of the button before exporting — all generated files will use this name as a prefix. Six formats are available:

FormatContents
📊 Plots & CSV (ZIP)All visible plots as PNG images + results table as CSV, packaged in a ZIP archive.
📄 Single HTMLSelf-contained HTML file with the results table and all plots embedded as images. Can be opened in any browser without the application.
📊 Excel XLSXResults table in Excel format with formatted cells and column headers.
📄 PDFMulti-page landscape PDF: configuration page, results table, engine evaluation, and one plot per page.
📊 PowerPointPPTX presentation with one slide per plot, suitable for reporting.
📄 JSONFull raw PyRolL simulation output as a JSON file. Useful for post-processing or archiving.

7. Raw Data

At the bottom of the Results tab, a collapsible Raw Data panel is available. Click the Raw Data ▼ Show header to expand it.

The panel shows the complete PyRolL simulation output as formatted JSON, up to a preview of 500 lines. If the output exceeds 500 lines (which is typical for large rolling schedules with pillar data), a notice is shown:

Showing first 500 of N lines — use Export → JSON for full data.

The raw JSON contains all PyRolL pass objects with their full attribute set, including intermediate values that are not shown in the results table (e.g. individual pillar positions, sub-pass data, heat transfer coefficients).

Use Export → JSON to download the full untruncated raw data for scripting, comparison, or archiving in external tools.
