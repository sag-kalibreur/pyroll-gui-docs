Pass Design Guide

PyRoll GUI  |  Date: May 2026

1. Overview

The Pass Design tab is the central interface for defining a rolling schedule. A rolling schedule consists of a sequence of units — rolling passes, transport sections, cooling sections, and finishing blocks — that describe the complete path of the workpiece from the initial billet to the finished product.

Each unit in the table represents one processing step. Units are executed in order from top to bottom. The simulation passes the workpiece state (temperature, cross-section geometry, velocity) from one unit to the next.

1.1 Multiple Sequences

The Pass Design tab supports any number of sequences. Each sequence is an independent rolling schedule with its own pass table, solve method, and spreading model. New sequences are added with the + Add Sequence button; existing sequences can be collapsed, reordered, or deleted individually.

Multiple sequences enable two important simulation modes:

ModeDescription
Semi-continuous rolling A multi-strand or tandem mill where different pass groups run at different conditions. Each strand is defined as a separate sequence.
Reversing mill The workpiece passes back and forth through the same stand. Each pass direction is modelled as a separate sequence with a mirrored or identical groove set.
Automatic profile transfer: After a sequence is solved, its output profile — including cross-section geometry, temperature, strain, and all other physical state variables — is automatically used as the input profile of the following sequence. The next sequence cannot be run until the previous one has been solved. Use ▶▶ Run All Sequences to solve all sequences in order in a single step.

1.4 Unit Row Structure

Every unit row in the schedule shows:

  • A type selector (left) to switch between unit types
  • An expand / collapse button to show or hide detailed parameters
  • A delete button to remove the unit from the schedule
  • The unit's parameter fields when expanded
Tip: Rows can be collapsed to keep the overview clean when working with long schedules. All parameters are preserved when a row is collapsed.

3. TwoRollPass

TwoRollPass

The TwoRollPass is the standard rolling pass. Two rolls with identical groove profiles are arranged symmetrically. The workpiece is reduced and shaped as it passes through the roll gap.

3.1 Parameters

FieldUnitDescription

GapmmDistance between the barrel surfaces of the two rolls (i.e. the clearance between the roll bodies outside the groove). A gap of 0 means the roll barrels are touching.
Define Velocity by—Choose whether to specify the roll surface speed directly (Velocity) or via the rotational frequency (Rotational Frequency).
Valuem/s or 1/sNumeric value corresponding to the velocity definition selected above.
Coulomb Friction Coefficient—Friction coefficient μ between workpiece and roll surface. Typical values: 0.2–0.4 for hot rolling.
Orientation—Roll axis orientation: horizontal (rolls above/below) or vertical (rolls left/right).
Groove Type—Cross-sectional profile of the roll groove. See Section 6 for available types. The i button next to the selector opens a preview panel showing all groove types with their geometric parameters.
Groove ParametersmmGeometry parameters specific to the selected groove type (radii, depths, widths). See Section 6. Use the Library button to load or save a groove directly from the Groove Library.
Groove Library: Inside the Groove Parameters box a 📖 Library button opens the Groove Library modal. From there you can:
  • Load — select a previously saved groove entry and fill all parameter fields automatically.
  • Save to Library — store the current groove parameters under a name for reuse across pass designs.
The Library button is available for TwoRollPass, AsymmetricRollPass (both upper and lower groove), and ThreeRollPass.

3.2 Roll Parameters

Expanding the Roll Parameters section reveals additional roll-specific settings:

3.3 Roll Pass Visualization

The Roll Pass Visualization panel shows a 2D cross-section of the groove caliber overlaid with the incoming workpiece profile. This allows quick visual verification that the groove fits the incoming cross-section without overfilling.

Note: In the Pass Design view, only the groove caliber outline is displayed — without the incoming workpiece profile. The full visualization including the workpiece cross-section overlay is available after running the simulation in the Roll Pass Plots and Overall Plots.

4. ThreeRollPass

ThreeRollPass

The ThreeRollPass represents a three-roll mill stand (e.g. Kocks or PSW mill) where three rolls are arranged at 120° intervals around the workpiece.

FieldUnitDescription

Define Velocity by—Surface velocity or rotational frequency (same as TwoRollPass).
Coulomb Friction Coefficient—Friction coefficient μ.

Groove Type—Groove profile (subset of types — round-type grooves typically used).

5. AsymmetricRollPass

AsymmetricRollPass

An AsymmetricRollPass uses two rolls with different groove geometries (upper and lower). A common example is a pass where one roll has a shaped groove (e.g. SplineGroove) and the other is flat (FlatGroove).

For details on the simulation method, see the dedicated help page: Asymmetric Roll Pass – Method & Results.

6. Transport

Transport

A Transport unit models the free transport of the workpiece between two stands — for example, on a roller table or a runout table exposed to ambient air. During transport, the workpiece loses heat to the surrounding environment.

6.1 Parameters

FieldUnitDescription
Label—Unique name for this transport section.
Define by—Specify the transport section by physical length (m) or elapsed duration (s).
Valuem or sNumeric value corresponding to the definition selected above.
Environment Temperature°CAmbient air temperature surrounding the workpiece. Typical value: 20 °C.
Heat Transfer CoefficientW/(m²·K)Convective heat transfer coefficient between the workpiece surface and the environment. Typical range for free air convection during hot rolling: 10–30 W/(m²·K).

6.2 Thermal Calculation

PyRolL models the temperature evolution during transport using Newton's law of cooling. The surface heat flux \( q \) is:

$$q = \alpha \cdot (T_\text{surface} - T_\text{environment})$$

where \(\alpha\) = heat_transfer_coefficient [W/(m²·K)], \(T_\text{surface}\) = workpiece surface temperature, \(T_\text{environment}\) = environment_temperature.

If Define by length is selected, the transit time is internally derived from the exit velocity of the preceding roll pass and the specified length. If Define by duration is selected, the transit time is used directly.

Note: Transport units do not perform any deformation — the cross-section geometry is unchanged. Only the workpiece temperature is updated.

7. CoolingPipe

CoolingPipe

A CoolingPipe unit models a controlled water cooling section, such as a water box or quench station. The workpiece passes through a pipe with flowing coolant, resulting in significantly stronger heat extraction than free-air transport.

7.1 Parameters

FieldUnitDescription
Label—Unique name for this cooling section.
Define by—Specify by length (m) or duration (s) — same logic as Transport.
Valuem or sLength or duration of the cooling section.
Inner RadiusmmInner radius of the cooling pipe. Determines the annular gap between pipe wall and workpiece.
Coolant Temperature°CInlet temperature of the cooling water. Typical value: 15–25 °C.
Coolant Volume Fluxm³/sVolumetric flow rate of the cooling water through the pipe. Controls the intensity of forced convection.

7.2 Difference to Transport

Unlike a Transport unit (air cooling, low heat extraction), a CoolingPipe uses forced convection with water, resulting in heat transfer coefficients typically 1–3 orders of magnitude higher. This is relevant for controlled cooling of wire rod and bar products after the finishing block.

Typical use case: Stelmor line cooling after a wire rod finishing block — alternating CoolingPipe sections and Transport (air cooling) sections to control the metallurgical transformation temperature.

8. FinishingBlock

FinishingBlock

A FinishingBlock is a mechanically coupled group of roll passes driven by a single motor. All sub-passes within the block are kinematically linked: the roll surface velocity of each sub-pass is determined by the shared motor RPM and the individual gear ratio of each stand.

8.1 Block Parameters

FieldUnitDescription
Block Name—Label for the entire finishing block (e.g. Finishing Block).
Motor—Selection of the drive motor from the motor library. Determines the available power and maximum torque.
Motor RPMrpmRotational speed of the drive motor. This single value controls the velocity of all sub-passes through their respective gear ratios.

8.2 Sub-Pass Parameters

Each sub-pass within the FinishingBlock is a TwoRollPass with the following differences:

  • No individual velocity input — velocity is derived from the block motor RPM
  • An additional Gear Ratio field per sub-pass
FieldUnitDescription
Label—Unique name for this sub-pass (auto-generated as FB1, FB2, …)
Gear Ratio—Transmission ratio \(i\) between motor shaft and this roll stand. The rotational frequency at the rolls is \(f = n_\text{motor} / (i \cdot 60)\).
GapmmDistance between the barrel surfaces of the two rolls.
Groove Type / Parameters—/mmSame as for a standalone TwoRollPass (see Section 3).

8.3 Velocity Calculation

The roll surface velocity for each sub-pass is computed as:

$$v_\text{roll} = \frac{n_\text{motor}}{i \cdot 60} \cdot 2\pi \cdot r_\text{roll}$$

where \(n_\text{motor}\) = motor RPM,  \(i\) = gear ratio,  \(r_\text{roll}\) = roll radius [m].

The displayed rotational frequency (shown in the UI next to each gear ratio input) is: \(f = n_\text{motor} / (i \cdot 60)\) [1/s].

Note: The GUI additionally computes and displays the inter-stand tension / compression stress between consecutive sub-passes within the FinishingBlock, based on the resulting velocity and cross-section profile.

8.4 Transport-Between Elements

Between sub-passes inside a FinishingBlock, optional Transport-between elements (shown with a yellow border in the UI) can be added to model short transport sections between individual stands within the block — for example, a short air gap or guide tube. These follow the same logic as standalone Transport units (see Section 6).

8.5 Adding and Removing Sub-Passes

  • Add Sub-Pass — appends a new sub-pass at the end of the block
  • × (Remove) button on each sub-pass row — removes that sub-pass
  • Delete Block — removes the entire FinishingBlock from the schedule
Note: The FinishingBlock motor power check is performed during simulation. If the required power exceeds the motor's rated power, a warning is shown in the results.

9.1 FlatGroove

A FlatGroove has a perfectly flat bottom (no curvature). It is used for flat rolling passes or as one side of an asymmetric roll pass (see Section 5). Only the groove width is specified.

9.2 SplineGroove

A SplineGroove is a custom free-form groove defined by a set of control points. This type is used when none of the standard geometric profiles accurately represents the actual roll caliber. The control points are entered as a list of (y, z) coordinate pairs describing the groove contour.

9.3 ConstrictedCircularOvalGroove

A variant of the CircularOvalGroove with an additional constriction (reduced width at the groove exit plane), used to improve material confinement. No separate SVG is available; the shape is similar to CircularOvalGroove with narrower flanks.

10. Solve Methods

The Solve Method defines how PyRolL distributes and iterates the velocities across the pass schedule. The selector is located in the toolbar area above the pass table, next to the Run Simulation button.

MethodDescriptionAdditional Input
Standard Solve PyRolL's default solver. The workpiece is processed sequentially pass by pass. Velocities are determined implicitly by the continuity equation and the roll surface speed of each pass. No additional parameters required.

Note: The GUI additionally computes and displays the inter-stand tension / compression stress between consecutive passes based on the resulting velocity and cross-section profile.		 —
Forward Solve The simulation starts from a known entry velocity at the first pass. All subsequent pass velocities are computed forward using mass continuity. Useful when the inlet speed is known (e.g. from a roughing mill exit). Initial Velocity [m/s]
Backward Solve The simulation works backwards from a defined exit condition — the target final velocity. PyRolL adjusts the entire velocity profile so that the last pass exits at the specified speed. The final cross-section area is determined automatically from the schedule geometry. Final Velocity [m/s]

10.1 Velocity Continuity

All three methods respect the law of mass continuity between passes:

$$v_1 \cdot A_1 = v_2 \cdot A_2$$

where \(v\) = workpiece velocity [m/s] and \(A\) = cross-section area [m²] at the exit of passes 1 and 2 respectively.

10.2 Velocity Seed Requirement

For Forward Solve and Backward Solve, every roll pass in the schedule must have a positive velocity seed value (> 0). This allows the solver to establish the velocity ratios between stands before iteration. A value of 1 m/s is sufficient as a seed — the solver will scale it automatically.

Note: FinishingBlock sub-passes derive their velocity from the motor RPM and gear ratio. No manual velocity seed is needed for sub-passes — the block motor RPM serves as the velocity constraint.

11. Spreading Models

Lateral spreading describes the increase in workpiece width during rolling caused by material flowing sideways (in addition to elongation). PyRolL GUI offers three empirical spreading models, selectable in the toolbar area of the Pass Design tab.

The spreading model is a global setting for the entire sequence — it applies to all roll passes in the schedule. Default: Wusatowski.

11.1 Wusatowski Spreading

The Wusatowski model (1969) is the default spreading model. It describes lateral spreading through an empirical power law based on the reduction ratio and groove geometry:

$$\frac{b_1}{b_0} = a_T \cdot a_v \cdot a_M \cdot a_f \cdot \left(\frac{h_0}{h_1}\right)^{\,\psi}$$

where \(b_0, b_1\) = workpiece width before/after the pass, \(h_0, h_1\) = height before/after, \(\psi\) = Wusatowski exponent (function of roll diameter, workpiece dimensions and reduction), and \(a_T, a_v, a_M, a_f\) = correction factors for temperature, velocity, material, friction.

The exponent \(\psi\) is determined empirically and accounts for roll diameter, initial dimensions, and the degree of reduction. Wusatowski is well-suited for conventional groove rolling (oval, round, box sequences).

Velocity correction \(a_v\): Wusatowski explicitly includes a rolling-speed correction factor: $$a_v = \bigl(-0.002958 + 0.00341\,\varepsilon\bigr)\cdot v \;+\; 1.07168 - 0.10431\,\varepsilon$$ where \(\varepsilon = (h_0-h_1)/h_0\) is the relative height reduction and \(v\) the roll surface velocity in m/s. The roll velocity entered in the pass parameters therefore directly affects the calculated spreading with the Wusatowski model.
Friction — no effect with Wusatowski: The internal friction correction factor is fixed at \(a_f = 1\). The Coulomb Friction Coefficient entered in the pass parameters has no influence on the Wusatowski spreading result. It is only used by other sub-models (force, torque, gripping analysis).

11.2 Marini Spreading

The Marini model (1979) extends the power-law approach with additional consideration of the groove caliber geometry. It provides improved accuracy for passes with strong lateral confinement (e.g. box and diamond grooves) where the groove walls actively constrain the spreading.

The core of the Marini model is the first Marini parameter:

$$a = \frac{\sqrt{\Delta h_{eq}}}{2\,\mu\,\sqrt{R}}$$

where \(\Delta h_{eq}\) = equivalent height reduction, \(\mu\) = Coulomb friction coefficient, \(R\) = working roll radius.

Friction — strong effect with Marini: The Coulomb friction coefficient \(\mu\) appears directly in the denominator of the first Marini parameter. A higher \(\mu\) reduces \(a\) and therefore reduces the spreading correction term \(\tfrac{1+a}{1-a}\). The friction value entered in the pass parameters has a significant influence on the Marini spreading result — in contrast to Wusatowski.
Velocity — no effect with Marini: The Marini formula contains no rolling-speed term. The roll velocity entered in the pass parameters has no direct influence on the Marini spreading result.

11.3 Hill Spreading

The Hill model is based on the analytical theory of plasticity (R. Hill, 1950) and derives spreading from the plane-strain deformation assumption. It is less empirical than Wusatowski and Marini, but typically gives good results for flat and near-flat passes where the plane-strain assumption is approximately valid.

11.4 Choosing a Model

Tip: When calibrating a pass schedule against measured data, compare the Elongation Spread chart in the results tab for each model — it directly shows the spreading vs. elongation relationship per pass.

12. Tips and Common Mistakes

10.1 Label Uniqueness

Every unit in the schedule must have a unique label. Duplicate labels cause a validation error before the simulation starts. The error message shows which labels are duplicated.

10.2 Velocity Seed for Forward/Backward Solve

When using the Forward/Backward solve method, every roll pass must have a velocity value greater than zero. A velocity of 0 will prevent the solver from converging.

10.3 Roll Temperature

The Roll Temperature field in the roll parameters must always be filled in. Leaving it empty causes a validation error. Typical roll temperatures in hot rolling: 60–150 °C depending on cooling intensity and stand sequence.

10.4 Transport Length vs. Duration

If Define by length is chosen for a Transport unit, the simulation requires the exit velocity from the preceding roll pass to compute the transit time. Ensure the preceding pass has a valid velocity. If no preceding roll pass exists (e.g., the first unit is a Transport), use Define by duration instead.

10.5 FinishingBlock — Gear Ratio Direction

A gear ratio \(i > 1\) means the roll rotates slower than the motor; \(i < 1\) means faster. For a typical finishing block with increasing roll speed from stand to stand (to compensate for cross-section reduction), gear ratios decrease from the first to the last sub-pass.
